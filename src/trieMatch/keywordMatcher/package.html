(keyword lookup)    The KeywordMatcher package implements several extremely powerful and efficient keyword matches, including keymatch, in-order matching, prefix matching and weight-based aggregation of multiple matches. It is based on lexicographic tries. This structure is related to the suffix tree (see Vittier) gives big advantages in search quality as well as performance. The basic algorithm is to sort the query terms and perform a greedy search over the keymatches (these are also sorted). By retaining the index arrays to the original order, we can recover the sequence and position data from the results of searching in "sort space". This gives a log log speedup over other algorithms, but (more) importantly it also gives a methodology for aggregation of multiple submatches.

These capabilities are organized into interfaces and methods, with support for various flavors including:

Valuation / Scoring
 Tokens, term matches, length and positions
 Aggregation / Combining
  AND, ANDacc, ANDaccN, OR
Matching / Collection
  All, longest, top, prefix
 Storage / Retrieval
  Lexicographicall, prefix trie
